## Project Overview

Simplefysed Solutions marketing website built with Next.js 16, React 19, and TypeScript. Features interactive 3D graphics and advanced animations; the hero, header, footer, productized-solutions section (`ServicesSection`), tech stack slider, the studio section (`components/studio/`: a Paper & Ink 3D photo-studio where the SIMPLEFYSED nameplate drops letter by letter; uses three's own TextGeometry, NOT drei Text3D, which hangs the GPU on three r182), and the `/solutions`, `/about`, `/linktree`, and `/case-studies` pages use the "Paper & Ink" editorial design (see `DESIGN_PROFILE.md`). The case-studies listing and `[slug]` detail were migrated to Paper & Ink (2026-07-07): their content is consolidated in `src/data/caseStudies.ts` and both routes are now server components (`generateStaticParams`/`generateMetadata`/`notFound()`) that render client sections in `components/case-studies/`. Legacy neon was removed 2026-07-08 (tokens + glow utilities from `globals.css`, the dead `CTASection`/`ServicesShowcase`/`ui/Card`/`GlowEffect`/`GradientText`/`ImageLightbox`/`Logo` components, `Button`'s neon variants, and the old Three.js hero `HeroCanvas`+particle scenes); the site UI is now fully Paper & Ink, the one remaining old-neon spot being the transactional email HTML in `api/contact/route.ts`. The studio section's 3D CTA labels are the home page's closing actions. Paper & Ink sections set `data-surface="paper"` for on-theme selection/focus styles from `globals.css`; the footer is the one inverted section (paper text on an ink field) and sets `data-surface="ink"`. The studio's font subset (`inter-semibold.typeface.json`) is regenerated by `website/scripts/generate-typeface.mjs`; read that script's header before touching it.

## Architecture

### Directory Structure (`website/src/`)

- **app/** - Next.js App Router pages
  - `page.tsx` - Home page
  - `solutions/` - Productized solutions detail page (Paper & Ink; anchors per solution slug)
  - `case-studies/` - Case studies listing and `[slug]` dynamic routes
  - `privacy/` and `terms/` legal pages

- **components/** - Organized by feature:
  - `hero/` - Home hero: `HeroSection` (static isometric image) + `HeroContent`, Paper & Ink
  - `layout/` - Header, Navigation, MobileMenu, Footer
  - `sections/` - Home-page sections (`ServicesSection`, `TechLogoSlider`)
  - `solutions/`, `about/`, `case-studies/`, `studio/`, `contact/`, `linktree/` - Paper & Ink feature components
  - `ui/` - Reusable components (`Button` with `ink`/`inkOutline`/`rust` variants)

- **data/** - Shared content data (`solutions.ts`: the three productized solutions, consumed by the home `ServicesSection` and the `/solutions` page)
- **hooks/** - Custom React hooks (useMousePosition)
- **stores/** - Zustand stores (contactStore.ts)
- **lib/** - Utilities (`cn` for classnames, `sleep`, `lerp`, `clamp`, `randomBetween`)
- **styles/** - Global CSS with design system variables

### Key Technologies

- **3D Graphics:** Three.js + @react-three/fiber + drei + postprocessing
- **Animation:** Framer Motion (primary), GSAP, Lenis (smooth scroll)
- **State:** Zustand (contact modal state)
- **Styling:** Tailwind CSS 4 with CSS custom properties for design tokens
- **Icons:** Lucide React

### Design System (Paper & Ink)

The site uses the **"Paper & Ink" editorial design** (full spec: `DESIGN_PROFILE.md`). Tokens live in `src/styles/globals.css`:

- **Palette:** `paper` `#F3F0E9` (page/section bg), `paper-card` `#FBF9F4` (raised surfaces/cards), `ink` `#1B1A17` (text, hairlines, dark buttons), `ink-muted` `#6B6760` (secondary text/labels), `rust` `#D6442F` (the single accent). Prefer alpha tints (`border-ink/10`, `bg-ink/25`, `text-ink/30`, `border-paper/15`) over new tokens.
- **Type:** `font-serif` = Playfair Display, display/headlines ONLY (`font-medium tracking-[-0.02em]`, tight leading, roman with one `italic text-ink-muted` line mixed in). `font-sans` = Inter, all body/UI (default). `font-mono` = JetBrains Mono, kickers/metrics/micro-labels (`uppercase` + wide tracking).
- `globals.css` holds Paper & Ink tokens only. The legacy neon token system (`--bg-*`, `--neon-*`, `--text-*`, `--glow-*`) and glow/skeleton utilities were removed 2026-07-08.

**Section shell & layout**
- Base `::selection`/`:focus-visible` in `globals.css` are on-brand (rust selection, ink focus), so the whole page is Paper & Ink by default. The footer is the one inverted section (`data-surface="ink"`, `bg-ink`, paper text), which flips selection/focus to paper. Sections still carry `data-surface="paper"` as a semantic marker.
- Standard shell: `border-t border-ink bg-paper px-8 sm:px-12 lg:pl-32 lg:pr-24 xl:pl-40`. The oversized **asymmetric left inset** is the editorial signature. Full section seams are `border-t border-ink`; internal dividers are `border-ink/10` hairlines; square corners (`rounded-none`) on buttons/tabs/cards.
- **Fill the full content width** with grids (two-column spreads, 12-col registers, dotted-leader rows). Do NOT trap content in a narrow `max-w-2xl` left column (it leaves the right half empty). Centered comps (`AboutCompany`, the case-study detail hero) instead drop the inset for symmetric `px-6 sm:px-8` + `items-center text-center`.
- **Accent discipline:** rust appears ONCE per logical unit (the payoff word, or the single primary button). Never two rust elements adjacent.
- **Motion:** `fadeUp` = `{opacity:0, y:24}` → `{opacity:1, y:0}`, duration `0.55`, ease `[0.22, 1, 0.36, 1]` (used everywhere). Scroll reveals use `whileInView viewport={{ once: true }}`; above-the-fold hero content uses `animate` (immediate) with staggered `delay`. A `whileInView` block renders blank in a Playwright `fullPage` screenshot (rAF-throttle artifact, not a bug) — scroll it into view or force `opacity:1` to verify.

### Buttons & Actions
- **Primary (marketing) = rust fill:** `<Button variant="rust" className="rounded-none px-[30px] py-4 text-[15px] font-semibold">` (e.g. "Start a Project", case-study "Visit"). For a link, style an `<a>`/`<Link>` as `rounded-none bg-rust px-[30px] py-4 text-[15px] font-semibold text-paper hover:bg-rust/90`.
- **Secondary = ink text-link:** `text-[15px] font-semibold text-ink border-b-[1.6px] border-ink hover:text-rust hover:border-rust`, with a lucide arrow that slides on hover (e.g. "Schedule a call", "Back to Case Studies"). Pair one fill + one text-link per row; never two fills.
- **Utility/nav = ink fill:** `<Button variant="ink">` (e.g. header "Get Started", listing "Learn More").
- **In-page scroll jumps** ("Learn More" on the listing, "Read More" on a detail with no live link): a plain `<button onClick>` calling `document.getElementById(id)?.scrollIntoView({ behavior: 'instant', block: 'start' })` on a section with `id` + `scroll-mt-24` (clears the fixed 64px header). Do NOT use a `#hash` `<Link>` for this — next/link and `HashScroll` both fire and overshoot.
- The contact modal opens via `useContactStore((s) => s.openModal)`.

### Copy Guidelines
- **Never an em dash** (`—`); restructure into two sentences, a comma, a colon, or parentheses. No exclamation marks, no superlatives; calm, confident, first-person-plural.
- Simplefysed is a **"software company"** (or "software and AI engineering company"), never studio / agency / firm. Founder is **Vin Mangelsdorf** (never "Mangels").
- **House metric format:** monospace value + sentence-case `uppercase` label; comma thousands (`3,876`), tilde approximations (`~30`), spaced-slash rates (`Tasks / month`). **Title Case** for actions/nav; sentence case for stat labels.
- Case-study detail subtitles (`tagline` in `caseStudies.ts`) are short, precise, curiosity-driven, and set in readable **sans** (not serif italic). The longer `description` feeds the listing card and OG/social meta.

### Reusable Paper & Ink Motifs
Compose these instead of inventing layout (reference implementations in parens):
- **Flanked masthead kicker** — a mono label with `h-px w-10 bg-ink/25` rules either side (`AboutCompany`, case-study detail hero).
- **Metrics ledger** — a row of `{value,label}` mono cells (`AboutCompany`, home `HeroContent`).
- **Folio bar** — a mono label pair between `border-y border-ink/10` rules (`AboutFolio`; case-study "The Brief"/"The Outcome").
- **Drop cap** — `first-letter:` utilities on an opening paragraph (`AboutFounder`, case-study Challenge).
- **Registration-mark corner brackets** around a plate (`AboutFounder` portrait, `linktree`).
- **Dotted-leader index** — label → `flex-1 border-b border-dotted border-ink/25` → mono page/category (`Footer`, case-study "More Case Studies").
- **Full-width editorial bodies** via two-column spreads or 12-col registers (`AboutHowWeOperate` 3/5/4; case-study "The Brief" Challenge|Solution spread).

### Code Patterns

- Use `'use client'` directive for interactive components
- Path alias: `@/*` maps to `./src/*`
- Component variants use the `cn()` utility combining clsx + tailwind-merge
- Framer Motion `whileInView` for scroll animations, `whileHover`/`whileTap` for interactions
- Hero sections must always span exactly one browser height: `flex min-h-dvh flex-col` on the section, `flex-1` plus centering (`content-center` on a grid, `justify-center` on a flex column) on the main block, so the closing band (pull quote, contents nav) pins to the fold and the next section's `border-t` lands exactly at the bottom of the first screen. On viewports shorter than the content the hero keeps its natural height and scrolls. When designing or changing a hero, verify at multiple viewport heights (about 864, 1000, and 1200 px); see `AboutCompany` and `SolutionsHero` for the reference implementation.
